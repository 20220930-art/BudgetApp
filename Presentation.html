Project Overview

BudgetApp helps users track income, expenses, budgets, and savings goals.

Core Features

  • User registration and login
  • Add income and expense transactions
  • Create monthly budgets per category
  • Budget alerts when nearing or exceeding limits
  • Savings goals with progress tracking
  • Reports and analytics by category
  • Notification system for budget alerts

SOLID Principles

1. Single Responsibility

Each route file handles one thing. authRoutes.js = login/signup only. transactionRoutes.js = transactions only.

2. Open/Closed

New report or notification types added without changing existing code.

3. Dependency Inversion

All models depend on PgPool abstraction. Switch databases = change one file.

Challenges and Lessons

  • Learning Alpine.js reactivity model
  • PostgreSQL connection pooling with pg

  • JWT token handling in browser

Lessons Learned

  • MVC makes code easier to maintain
  • SOLID keeps codebase organized
  • Design patterns solve real problems
  • Good DB schema design is crucial
  • Planning diagrams before code saves time
